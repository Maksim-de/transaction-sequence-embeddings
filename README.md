# Эмбеддинги последовательностей транзакций

Репозиторий с кодом и ноутбуками к ВКР **«Исследование методов обучения эмбеддингов для последовательностей транзакций»**.

Проект посвящен исследованию представлений для финансовых транзакций и пользовательских историй операций. Основной фокус:

- иерархическая трансформерная архитектура для транзакций и их истории;
- разные способы учета числовых признаков;
- MLM-предобучение на транзакционных данных;
- downstream-задача детекции мошенничества;
- сравнение с сильным табличным бейзлайном на `CatBoost`.

Репозиторий остается исследовательским и notebook-first, но структура приведена к более понятному виду, чтобы по GitHub было легче понять, где лежит основной эксперимент, где абляции, а где подготовка данных.

## Структура репозитория

- `notebooks/core/TabBert_experiments.ipynb`  
Основной ноутбук с базовой иерархической моделью и ключевыми экспериментами по ВКР.
- `notebooks/variants/TabBert_numeric_concat.ipynb`  
Вариант, где числовой признак не проходит через полную иерархию, а конкатенируется перед классификатором.
- `notebooks/variants/Tabbert_without_history.ipynb`  
Альтернативный вариант без отдельного history-энкодера в исходном виде эксперимента.
- `notebooks/baselines/catboost_fraud_detection.ipynb`  
Классический бейзлайн на `CatBoost` для детекции мошенничества.
- `notebooks/data_preparation/data_preparation.ipynb`  
Ноутбук для подготовки и проверки данных.
- `docs/EXPERIMENT_COVERAGE.md`  
Краткая ревизия того, как эксперименты из ВКР соотносятся с файлами в репозитории.
- `requirements.txt`  
Минимальный список зависимостей, восстановленный по импортам из ноутбуков.

## Какие эксперименты есть в репозитории

В работе сравниваются:

1. Базовая иерархическая модель с отдельной числовой веткой.
2. Вариант с конкатенацией числового признака перед классификатором.
3. Дискретизация числового признака на бины.
4. Классический бейзлайн `CatBoost`.
5. Сравнение иерархической и альтернативной архитектуры.
6. Сравнение обучения с MLM-предобучением и без него.
7. Влияние длины окна истории.

Важно: эксперименты с окнами длины `5`, `10` и `15` выполнялись в `notebooks/core/TabBert_experiments.ipynb`. Сейчас в ноутбуке сохранен вывод только последнего запуска, поэтому не все результаты видны как отдельные сохраненные output-блоки, хотя сами секции и логика экспериментов относятся к одному и тому же файлу.

## Данные

Сами датасеты в этот репозиторий не входят. Данные берутся из оригинальной статьи TABULAR TRANSFORMERS FOR MODELING MULTIVARIATE TIME SERIES ([https://arxiv.org/pdf/2011.01843](https://arxiv.org/pdf/2011.01843))

При запуске ноутбуков пути к данным при необходимости нужно подстроить под локальное окружение.
